金币项目所开发的神秘海域系统，出现了一个结构体内一个byte类型的变量被异常置0的情况。
首先，是在给一个模块的结构体数据清空时。
用常用的 testStruct = &model{
    text : make(map[int]int)    
}方式给该结构体指针进行数据的清空
但是该方式并没有将testStruct实际指向的地址内的值给清空，而是又申请了一块新的空间，让testStruct指向了这块空间
也就是说原有的testStruct所指向的地址的值并没有修改
然后改成 *testStruct = model {
    text : make(map[int]int)
     num : 0  (这里不加，也会赋0值)
}
将原有的testStruct结构体指针解引用，取地址内的值，然后结构体初始化进行修改。
但是发现这种写法还是会将未操作的整型值赋为0值。
总结：如果在给结构体指针进行清空数据时，还是要单独给结构体内对应的字段进行操作修改，不可整体初始化
